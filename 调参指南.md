# 橙色线圈检测 — 调参指南

启动 `uv run streamlit run streamlit_app.py` 后浏览器自动打开 <http://localhost:8501>，左侧上传图片，下面三个折叠面板共 19 个滑动条。**改任何滑动条都会自动重跑算法**，不用按按钮。

**重要：本指南所有滑动条名称都跟界面 label 完全一致**，照着找就行。

> 部署到 Streamlit Cloud 的版本访问 <https://auto-detect.streamlit.app/>，行为一致。

---

## 处理流程速览

```
原图 (BGR)
  └─ 转 HSV
       ├─ 板整体 HSV → mask_raw → 切出两块板的矩形 (rects)
       └─ 橙色 HSV → mask_coil
            └─ ROI 内: 开运算 → 闭运算 → mask_coil_filled
                 └─ Canny 边缘 ∩ mask → 找轮廓 → 面积过滤 → 画绿框
```

简单说：先用"板"的颜色框出两块基板的位置，再在板内部用"橙色"的颜色筛出线圈区域，最后用 Canny + 面积阈值挑出真正的线圈轮廓画绿色。

---

## 一、「板整体 HSV 范围」面板（默认折叠）

**作用**：定位两块基板的外接矩形。这一步**只决定绿色大矩形框画在哪**，不直接影响轮廓检测的细节。

| 界面滑动条 | 默认 | 范围 | 调大/调小的效果 |
|---|---|---|---|
| **板 H min（色相下限）** | 8 | 0~179 | 调小 → 把更红/紫的区域也算进来；调大 → 收紧到更纯的橙黄 |
| **板 H max（色相上限）** | 30 | 0~179 | 调大 → 把黄绿也算进来；调小 → 收紧到更红的橙 |
| **板 S min（饱和度下限）** | 50 | 0~255 | 调小 → 灰色/低饱和也进来（板更大）；调大 → 只要鲜艳颜色 |
| **板 S max（饱和度上限）** | 255 | 0~255 | 一般保持 255 |
| **板 V min（明度下限）** | 50 | 0~255 | 调小 → 暗部也进来；调大 → 只要亮的（光线暗时往下调） |
| **板 V max（明度上限）** | 255 | 0~255 | 一般保持 255 |

**何时调**：
- 信息栏报"未检测到板" → 把 H 范围拧宽，或把 S/V 下限调小
- 板矩形画得太大（圈到背景）→ 收紧 H 范围或抬高 S 下限

---

## 二、「橙色线圈 HSV 范围」面板（**最常调的一组**）

**作用**：从板内部筛出"橙色线圈"区域，生成 mask，后续 Canny 只在这里找边。

比"板整体"窄很多——要的是**高饱和、亮的纯橙**。

| 界面滑动条 | 默认 | 范围 | 调大/调小的效果 |
|---|---|---|---|
| **线圈 H min（色相下限）** | 10 | 0~179 | 调小 → 偏红橙也算（误检红色干扰）；调大 → 只要纯橙黄 |
| **线圈 H max（色相上限）** | 28 | 0~179 | 调大 → 偏黄绿也算；调小 → 只要更红的橙 |
| **线圈 S min（饱和度下限）** | 120 | 0~255 | **调大 → 只要更鲜艳的橙（最常用来去噪）** |
| **线圈 S max（饱和度上限）** | 255 | 0~255 | 一般保持 255 |
| **线圈 V min（明度下限）** | 90 | 0~255 | **调大 → 只要更亮的橙（去暗部噪点）** |
| **线圈 V max（明度上限）** | 255 | 0~255 | 一般保持 255 |

**何时调**：
- 绿色小点太多 → **线圈 S min** 120→160~180、**线圈 V min** 90→120~140
- 漏检线圈（线圈区域被画灰）→ H 范围拧宽，或 S/V 下限调小
- 把背景里偏色的东西误识别成线圈 → 收紧 H 范围

---

## 三、「Canny / 形态学 / 面积」面板

### 1. Canny 边缘检测

**作用**：在 mask 内部找边缘——线圈的圈线就是靠这一步抠出来的。

| 界面滑动条 | 默认 | 范围 | 调大/调小的效果 |
|---|---|---|---|
| **Canny low（边缘弱阈值）** | 30 | 0~255 | 调大 → 弱边缘被抛弃（线条少且短，去噪用）；调小 → 弱边缘也保留 |
| **Canny high（边缘强阈值）** | 80 | 0~500 | 调大 → 只留最强边缘（线变稀疏）；调小 → 留下更多中等强度边缘 |

经验：**Canny high** 通常是 **Canny low** 的 2~3 倍。

**何时调**：
- 边缘太碎、绿点多 → 两个都调高（30/80 → 50~70 / 120~160）
- 线圈轮廓断断续续 → 两个都调低，或加大闭运算

### 2. 形态学

**作用**：清理 mask 上的毛刺、把断开的细线连成片。

| 界面滑动条 | 默认 | 范围 | 调大/调小的效果 |
|---|---|---|---|
| **开运算核（奇数，去小白点）** | 3 | 1~21 | 调大 → 抹掉更多小白点（去噪强）；过大会把细线圈也抹掉 |
| **闭运算核（奇数，连断口）** | 11 | 1~31 | 调大 → 把更远的断点连起来（线圈合并成块）；过大会粘连相邻线圈 |
| **闭运算迭代次数** | 3 | 1~8 | 调大 → 闭运算重复次数，效果叠加 |

**何时调**：
- 散点噪声多 → **开运算核** 3→5~7
- 线圈轮廓有断口、不闭合 → **闭运算核** 或 **闭运算迭代次数** 加大

### 3. 面积过滤（**消绿点最直接**）

**作用**：把太小或太大的轮廓直接丢掉。

| 界面滑动条 | 默认 | 范围 | 调大/调小的效果 |
|---|---|---|---|
| **最小轮廓面积（去小点）** | 30 | 0~5000 | **调大 → 小点全没（最常用去噪手段）**；过大会把真线圈也过滤掉 |
| **最大轮廓面积（去超大块）** | 6000 | 100~50000 | 调小 → 把整块大色斑过滤掉；通常不用动 |

**何时调**：
- 绿色小点太多 → **最小轮廓面积** 30 → 200~500，**优先调这个**
- 真线圈被过滤了 → **最小轮廓面积** 调小

## 调参 cheat sheet

### 现象 → 改哪个滑动条

| 现象 | 优先调 | 备选 |
|---|---|---|
| 绿点太多太碎 | **最小轮廓面积** 30→200~500 | **线圈 S min** / **线圈 V min** 调大；**开运算核** 调大 |
| 漏检线圈（应有的没框出来） | **线圈 H min** / **线圈 H max** 拧宽 | **线圈 S min** / **线圈 V min** 调小；**最小轮廓面积** 调小 |
| 线圈轮廓断断续续 | **闭运算核** / **闭运算迭代次数** 调大 | **Canny low** / **Canny high** 调小 |
| 提示"未检测到板" | 板的 6 个 HSV 全部拧宽 | 检查输入图片是否真有橙色 |
| 把红色/黄色误识别 | 收紧 **线圈 H min** / **线圈 H max** | 抬高 **线圈 S min** |
| 板矩形圈到背景 | 收紧 **板 H min** / **板 H max** | 抬高 **板 S min** |

### 推荐调参顺序

1. **先把板框出来**：调"板整体 HSV"面板的 6 条，直到信息栏不再报"未检测到板"，且两个绿色大矩形位置正确
2. **再筛橙色**：观察绿点和漏检情况，调"橙色线圈 HSV"面板
3. **最后清理**：用 **最小轮廓面积** 一刀切掉小点；不够再动 **开运算核** / Canny

### 保存与恢复

- **下载 params.json** 按钮 → 把当前 19 个滑动条的值打包下载到本地
- **上传 params.json 恢复参数** → 把之前下载的 JSON 拖回来，所有滑动条立即同步过去
- **恢复默认** 按钮 → 滑动条全部回到代码里的默认值
- 本地用户：把下载的 `params.json` 直接放回 `streamlit-coil-detect/` 目录，下次 `uv run streamlit run streamlit_app.py` 启动就自动加载

> 为什么不直接写文件？Streamlit Cloud 的容器是临时文件系统，写盘没意义（重启就丢、也不会回流到 git 仓库）。下载/上传方案在 Cloud 和本地都能用，而且可以管理多套配置。

## 大图怎么办（4K / 7K 输入）

代码里设了 `MAX_PROCESS_DIM = 2000`：上传的图最长边超过 2000 时，会先 `cv2.resize` 等比缩到 2000 再处理，主要原因有两个：

1. **Streamlit Cloud 内存只有 1GB**，原图 + HSV/gray/mask/edges/result 几个副本，4K 以上很容易 OOM
2. **默认参数（min_area=30、open_ksize=3 等）是按 ~2K 图调的**，自动降采样后这套阈值在任何分辨率下都能直接用，不用每换一张大图就重调一次

信息栏出现 `🔻 已降采样至 2000x1536 处理（坐标已还原到原图）` 就说明触发了降采样。返回的"匹配框中心"坐标和"图像尺寸"标签都是**原图坐标系**下的真实值，可以直接对应原图物理位置。

如果本地内存充足（16GB+）想全分辨率处理，启动前设环境变量：

```bash
COIL_MAX_DIM=8000 uv run streamlit run streamlit_app.py     # macOS/Linux
$env:COIL_MAX_DIM=8000; uv run streamlit run streamlit_app.py   # PowerShell
```

## HSV 色相速查（H 值范围）

OpenCV 的 H 通道是 0~179（不是 0~360）：

| 颜色 | H 大致范围 |
|---|---|
| 红 | 0~10 + 170~179 |
| 橙 | 10~25 |
| 黄 | 25~35 |
| 绿 | 35~85 |
| 青 | 85~100 |
| 蓝 | 100~130 |
| 紫 | 130~160 |
| 粉/品红 | 160~170 |

调"线圈 H min / H max"基本就在 **10~28** 这一段挪动。
